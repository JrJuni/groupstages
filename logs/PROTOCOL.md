# 📜 groupstages.com AI 에이전트 운영 프로토콜

> **이 파일은 모든 AI 에이전트가 처음 이 레포에 진입했을 때 가장 먼저 읽어야 할 운영 원칙입니다.**

---

## 프로젝트 정의

당신은 **groupstages.com** 프로젝트(2026 FIFA 월드컵 조별리그 경우의 수를 시나리오별로 보여주는 동적 웹 서비스)를 수행하는 AI 유닛입니다.

당신은 개별적인 작업을 수행하기 전과 후에 반드시 아래의 **[기록 및 참조 체계]**를 준수해야 합니다.

---

## 1. 작업 시작 전: Context 로딩 (Read)

모든 명령을 수행하기 전, 다음 파일들을 **순서대로** 읽고 현재 맥락을 파악하세요.

| 순서 | 파일 | 읽는 목적 |
|------|------|-----------|
| 1 | `logs/memory_user.md` | 사용자의 핵심 철학, 수학적 로직 기준, 디자인 원칙(North Star) 확인 |
| 2 | `logs/current_status.md` | 전체 진행 상황, 현재 활성 태스크, 블로커 확인 |
| 3 | `logs/memory_[agent_name].md` | 본인이 마지막으로 수행했던 실무의 기술적 디테일 확인 |

---

## 2. 작업 수행 중: 원칙

- **수학적 정확성**: 스포츠 시나리오의 경우의 수는 수학적 무결성이 최우선입니다.
- **최소 토큰 전략**: 불필요한 대화나 사소한 과정 기록을 지양하고, **'결정된 사항(Decisions)'**과 **'변경사항(Changes)'**에 집중합니다.

---

## 3. 작업 종료 후: 상태 동기화 (Write)

작업을 마친 후에는 반드시 다음 구조에 맞춰 파일들을 업데이트하세요.

| 파일 | 기록 내용 |
|------|-----------|
| `logs/current_status.md` | [공유 대시보드] 전체 로드맵 진행률(%), 현재 활성 작업, 다음 단계 신호 업데이트 |
| `logs/memory_[agent_name].md` | [실무 로그] 파일 수정 내역, 알고리즘 설명, 기술적 트러블슈팅 결과 등 |
| `logs/memory_user.md` | [철학 업데이트] 새롭게 확립된 방향성이나 우선순위가 있을 때만 사용자 승인 후 수정 |

---

## 4. 아카이빙 및 유지보수 규정

- **월간 이관**: 파일이 너무 길어져 가독성이 떨어질 경우, 지난 기록을 요약하여 `/logs/archive/memory_YYYY_MM.md`로 이동
- **스냅샷**: 아카이빙 시에는 반드시 해당 월의 핵심 성과와 최종 상태를 **5줄 이내**로 요약하여 현재 메모리 파일 상단에 남겨둠

---

## 파일 구조

```
/logs
  ├── /archive
  │     └── memory_YYYY_MM.md     # 과거 기록 (평소엔 읽지 않음)
  ├── PROTOCOL.md                 # 이 파일 — 운영 원칙
  ├── memory_user.md              # 사용자 가이드라인 (판단의 최상위 근거)
  ├── memory_[agent_name].md      # 에이전트별 개별 실무 기록 (기술적 디테일)
  └── current_status.md           # [중요] 전체 협업 상태 및 로드맵 (Shared State)
```

---

## 💡 새 에이전트 온보딩 가이드

새로운 에이전트가 합류했을 때:

1. 이 파일(`PROTOCOL.md`)을 먼저 읽는다
2. `memory_user.md` → `current_status.md` 순으로 컨텍스트를 로딩한다
3. 자신의 역할에 맞는 `memory_[agent_name].md` 파일을 생성하거나 이어서 기록한다
4. 작업 완료 후 반드시 `current_status.md`를 업데이트하고 커밋한다

> 이 레포에서는 **"로그를 확인해"라고 말하지 않아도** 에이전트가 자발적으로 파일을 열어본다.
