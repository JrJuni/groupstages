# i18n 다국어 작업 — 점검·개선 트래킹

> 다음 업데이트 시 한 번씩 검토하고 해결되면 ✅ 표시. 새 이슈 발견 시 추가.

## 우선순위 점검 항목

### 1. 보간 라벨 어순 (높음)
- [ ] **R32MatchupPreview**의 슬롯 라벨 — `${groupKey}조 ${rank}위`, `${groups.join('/')} 3위` 등은 한국어 어순에 맞춰져 있음. 영어("Group B 1st", "3rd from A/B/E/F/G")는 어순이 다름. 일본어/중국어도 어순 검증 필요.
- 해결: i18next 보간 + 언어별 템플릿 사용. 예) `t('scenario:slot.direct', { group, rank })` → ko: `"{{group}}조 {{rank}}위"`, en: `"Group {{group}}, {{rank}}{{ordinal}}"`
- 점검 방법: 각 언어로 전환해서 R32 매치업 미리보기 시각 확인.

### 2. RulesPage 본문 청크 단위 (중간)
- [ ] rules.json 키를 청크 단위(마크다운 문단 통째)로 잡았을 때 문단 안 인라인 강조/링크 처리 어색하지 않은지.
- 점검 방법: 각 언어 규칙 페이지 렌더 결과 비교.

### 3. 번역 품질 — 축구 용어 (높음)
- [ ] 헤드투헤드, 페어플레이, 골득실, 다득점, 추첨, 시드 등 각 언어 공식 표기 검증.
- [ ] FIFA 공식 사이트의 영어/스페인어/일본어/중국어 표기 참조.
- [ ] 일본어: "得失点差", "勝点", "ヘッドtoヘッド" → "直接対戦" 더 자연스러운지.
- [ ] 중국어: 簡體 vs 繁體 결정. 현재 zh = zh-CN(간체) 가정.
- 점검 방법: 사용자 검수 + 원어민 피드백.

### 4. 날짜 KST 표기 (중간)
- [ ] 모든 언어에서 "KST" 표기 유지가 자연스러운지. 일본/중국 사용자에게는 "한국 시간"이 어색할 수 있음.
- 대안: 사용자 시간대 자동 감지 + 토글로 KST/local 전환 옵션.
- 점검 방법: 일정 표시 영역 다언어 비교.

### 5. SEO hreflang (중간)
- [ ] `<link rel="alternate" hreflang="ko" href="..." />` 태그 5개 동적 주입 확인.
- [ ] Google Search Console에 sitemap 업데이트 (5개 언어 경로 모두 포함).
- 점검 방법: `view-source:` 로 head 확인 + Search Console 인덱싱 상태.

### 6. Cloudflare Pages SPA 폴백 (높음)

- [ ] 직접 진입(`/wc2026/es/scenarios`) 시 404 안 나는지.
- 점검 방법: 배포 후 각 언어 딥링크 직접 접속.
