## 핵심 철학

### 수학적 무결성이 최우선
- 경우의 수 계산에서 틀린 결과는 절대 허용되지 않는다
- FIFA 공식 규칙(승점 → 득실차 → 다득점 → 헤드투헤드 → 추첨)을 정확히 구현해야 한다
- 3위팀 8개 선별 기준도 동일한 규칙 적용

### 유연성 우선 (초기 개발 단계)
- 아직 초기 개발 단계이므로 유연함이 생명이다
- 빠른 실험 → 빠른 피드백 → 빠른 수정 사이클 유지
- 완벽한 설계보다 동작하는 결과물 우선

### API 연동 준비 병행
- 현재는 수동 입력이지만, API-Football (RapidAPI) 연동을 항상 염두에 두고 설계
- DB 스키마와 쿼리 구조는 API 응답 포맷과 호환되어야 한다

## 디자인 원칙

- **Dark mode** 기반 UI (FIFA 공식 색상 팔레트 참조)
- 모바일 반응형 필수
- 한국어 우선 UI (주 사용자층이 한국 축구 팬)
- 광고 슬롯(AdSense) 레이아웃 사전 반영

## 기술 스택 결정 사항

| 결정 | 이유 |
|------|------|
| React + Vite | 빠른 개발 속도 |
| Tailwind CSS | 유틸리티 클래스 기반 빠른 스타일링 |
| PostgreSQL (Replit 내장) | DB 연동 즉시 가능, 별도 설정 불필요 |
| Express REST API | 단순하고 명확한 엔드포인트 구조 |
| API-Football (RapidAPI) | 향후 실시간 데이터 소스 |
